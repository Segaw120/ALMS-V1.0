# ALMS: Autonomous Learning Memory System
## The Recursive Intelligence Engine (v2)

[![License: ALMS-PRL](https://img.shields.io/badge/License-ALMS--PRL-blue.svg)](LICENSE.md)
[![Axiomatic Deduction](https://img.shields.io/badge/Logic-Axiomatic--Deduction-green.svg)](#)
[![Parsimonious Intelligence](https://img.shields.io/badge/Ma--Principles-Parsimony-orange.svg)](#)

**ALMS** is a self-evolving, parsimonious intelligence engine designed to replace overreliance on traditional Machine Learning with a live symbolic memory network. Inspired by the principles of **Intelligence as Compression** (Yi Ma), ALMS treats learning not as weight adjustment, but as the reduction of experience into stable, systematically true **First Principles (Axioms)** that can be searched and reasoned over in real time.

---

### 🚀 Key Innovations

- **Parsimonious Memory**: Instead of memorizing noise, the system crystallizes data clusters into the simplest possible logical manifold.
- **Live Memory Reasoning**: The system searches memory dynamically at inference time and composes reasoning from stored experience rather than relying on a separate builder path.
- **Zero-Hallucination Mandate**: Every logical claim is strictly proven through citations of instructor-verified Axioms.
- **The Instructor Manifold**: A human-in-the-loop feedback system where an Instructor guides the evolution of the system's Axiom Genome.
- **Recursive Logic Refactoring**: Foundational changes to an Axiom trigger a recursive realignment of all dependent reasoning traces to ensure total consistency.
- **Hebbian Synaptic Field**: A living local vector field that strengthens important connections and prunes noise through temporal decay.

---

### 🏗 Architecture Overview

The system is structured into three primary layers:

1. **The Cerebellum (`engine/`)**: Handles the high-speed synaptic field, vault-watching, and the Pulse heartbeat.
2. **The Cortex (`logic/`)**: Orchestrates high-level reasoning, memory search, parsimonious compression, and instructor feedback loops.
3. **The Genome (`knowledge/`)**: A structured Obsidian-based Vault containing the system's identity, verified Axioms, and reasoning audit trails.

### 🚦 Getting Started

#### 1. Requirements
- Python 3.10+
- Obsidian (for managing the `knowledge/` vault)
- An LLM backend (local or API)

#### 2. Installation
```powershell
git clone https://github.com/Segaw120/ALMS-V1.0.git
cd ALMS-V1.0
pip install -r requirements.txt
```

#### 3. Running the Engine
Launch the unified Pulse and API server:
```powershell
python main.py
```

### 🎓 The Instructor Loop

As the **Instructor**, you guide the system's evolution by reviewing **Hypotheses** (`HYP-xxxx.md`) generated by the student.

- Use **`[[APPROVE]]`** to promote a hypothesis to a permanent **Axiom**.
- Use **`[[CORRECT]]`** to provide logical feedback and trigger a **Genome Mutation**.
- Use **`[[REJECT]]`** to prune noisy logical paths.

---

### 🧠 Live Memory Reasoning

ALMS is designed to reason directly from memory during execution.

- Query incoming tasks against the memory network.
- Retrieve the smallest relevant symbolic subset.
- Compose an answer or action plan from stored experience.
- Compare conflicting memories and resolve contradictions.
- Update confidence and persist useful traces for future retrieval.

This makes ALMS a live symbolic reasoning system, not just a memory builder.
